Control de Versiones

Controlador de versiones

Un controlador de versiones es un sistema que registra los cambios en un archivo o conjunto de archivos a través del tiempo para que se puedan recuperar versiones específicas de los mismos más adelante.

Control de versiones distribuido

Cuando los usuarios descargan la última versión, no solo están descargando los archivos; sino también realizando una copia de seguridad del repositorio.

¿Qué es GIT?

Git en pocas palabras es un controlador de versiones distribuido.

Instalación de GIT

Instalación en MacOS

Para realizar la instalación en este Sistema Operativo es recomendable tener instalado Homebrew. Haciendo uso del Terminal.app y de homebrew, escribimos el siguiente comando: brew install git

Instalación en Fedora y similares

Si nos encontramos en distribuciones como Fedora, podemos hacer uso del manejador de paquetes de dicha distribución. yum install git-core

Instalación en Debian y similares

Si nos encontramos en distribuciones basadas en Debian, tal como Ubuntu. apt-get install git

Instalación en Windows

Si nos encontramos el Windows la manera más sencilla de realizar la instalación es descargar el Instalador y seguir los pasos. https://git-scm.com/download/win

Ayuda en GIT

Para conocer la funcionalidad a detalle de un comando poder hacer uso del acceso al manual de usuario. man git config
O podemos hacerlo a través del acceso a la ayuda git config --help
Y si esto no fue suficiente ayuda, entonces la documentación en web https://git-scm.com/docs/

Configuración

Configurar en Git cual será el nombre de la persona que hará uso de él git config --global user.name "Pedro Perez"
Git hace uso del email del usuario cuando consolida la información, por esta razón necesitamos configurarlo git config --global user.email "pedroperez@ejemplo.com"
Revisar la configuración git config --list
También debemos conocer que Git guarda esta información dentro del archivo oculto .gitconfig en el Home del usuario.

Comandos básicos

Inicializar un repositorio
Un repositorio es una carpeta o conjunto de carpetas que contienen archivos. Podemos crear una carpeta y luego iniciar el repositorio dentro. mkdir Ejemplo && cd Ejemplo git init .
Agregar archivos al escenario (“stage”)
touch Archivo1.txt echo 'Hola Mundo' >> Archivo1.txt git add Archivo1.txt Al ejecutar el comando git add Archivo1.txt estamos confirmando (agregando el archivo al escenario) que los cambios que realizamos sobre Archivo1.txt se quieren respaldar la próxima vez que consolidemos la información.
Consolidar la información (“commitear”)
Para consolidar el archivo previamente creado y puesto en escenario. git commit -m "Commit Inicial" La bandera -m indica que se debe consolidar el archivo con un mensaje informativo.
